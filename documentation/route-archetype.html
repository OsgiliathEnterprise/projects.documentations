Purpose

The route module allow you to transform and route your messages see and read the Enterprise Integration Pattern In order to use it, generate the skeleton with the Osgiliath route archetype, then, create a simple route: The URL/entrypoints in bracket are from the cfg file of the feature module while executing on runtime and from the src/test/resources/cfg one when unit testing. Then, you’ve to reference this route on the camelContext section of the blueprint.xml file and in the src/test/resources/spring (for testing). You can also use @ContextName camel-cdi annotation and reference component in a configuration class For more informations, check the hello example on github
